# 実装計画（MVP）
## フェーズ1: 言語コア最小化
- 手書きパーサで `let` / `|>` / `fn` の最小サブセットを実装
- AST から実行可能な内部表現へ変換
- 名前解決と診断（シャドウイング禁止）を統合
## フェーズ2: 制御構文と安全境界
- `match` / `with` / `io do ... end` の導入
- 副作用を `io` 境界内に限定する評価ルールを実装
- エラー伝播の統一（Result ベース）
## フェーズ3: パイプラインと外部連携
- Stream ベースの `map` / `where` / `each` を実装
- `cmd` / `run` / `run_text` を接続
- JSON / Text 変換ブリッジの最小実装
## フェーズ4: 開発体験強化
- REPL 機能強化（複数行、簡易補完、履歴永続化）
- 診断メッセージ改善
- スナップショットテスト追加
## 完了条件
- 受け入れ基準 3 点（名前衝突、glob 非自動、引数分割なし）を常時満たす
- 最小スクリプトが REPL / バッチ双方で再現可能に動作する
